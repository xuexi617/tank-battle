# 坦克大战 - Battle City 🎮

一个 1990 年代风格经典的坦克大战像素游戏，使用纯 HTML/CSS/JavaScript 实现。

## 游戏说明

- **方向键 / WASD** 移动坦克
- **空格键 / J** 射击
- **手机端**：屏幕方向键 + FIRE 圆钮，触屏也能玩
- 保护你的基地（鹰旗），消灭所有敌方坦克

## 特性（当前版本：GBR504/tank-battle 精品覆盖版）

- 🎯 经典像素风格，多关卡地图（4 个关卡可选）
- 💥 砖墙 / 钢墙 + 多种地形障碍
- 👾 4 种敌方坦克类型，AI 各异
- 🌟 玩家出生无敌保护 + 防护盾道具
- 🔧 6 种道具（星 / 炸弹 / 时钟 / 铲子 / 头盔 / 加命等）
- 📱 桌面键盘 + 手机触屏双支持
- 🔊 Web Audio 合成音效（射击 / 爆炸 / 过关等）
- 🏆 最高分本地记录（localStorage）

## 如何游玩

**直接用浏览器双击打开 `坦克游戏.html` 即可游玩，无需安装任何依赖、无需 npm。**

> ⚠️ 本项目是纯单文件游戏，已移除 esbuild 构建链。请不要执行 `npm install` / `npm run serve`——那套构建残留会导致白屏，直接双击 `index.html` 最稳。

## 体验修复与增强（2026-07-10）

用户反馈两点：(1) 子弹打中敌人后敌人不消失、原地不动、随后不再刷敌、整体卡死；(2) 屏幕太小、节奏、地图内容。

### 1) 致命 Bug：击杀后卡死（已修复）
- **根因**：敌人被击杀只置 `e.alive=false`，但**从不从 `enemies` 数组移除**。后果有二：
  - 渲染层 `drawTank` 只对「死亡玩家」跳过绘制，死亡敌人仍被画出 → 表现为"打了不死、原地不动"。
  - `spawnEnemy` / `updateSpawns` 用 `enemies.length` 判断是否达到同屏上限（默认 4），**把尸体也算进去** → 尸体占满名额后永远不再刷敌；而 `enemyQueue` 消耗不完、通关条件 `enemyQueue===0 && 全死` 也迟迟不触发 → 表现为"没有敌人出现、一直不动"。
- **修复**（3 处精确改动）：
  - `update()` 末尾加 `enemies = enemies.filter(e=>e.alive);` —— 每帧剔除尸体（停止渲染、释放名额）。
  - `spawnEnemy` 与 `updateSpawns` 的刷敌闸门改为按**存活数**判断：`enemies.filter(e=>e.alive).length`。
- **验证**：强化 Node 桩，新增「强制连杀 8 次」用例。结果：击杀后敌人计数 3→2→1→0 正常递减（尸体被剔除），第 7 次击杀后 `enemyQueue` 由 21 降到 20（新敌正常补充），`state` 始终为 PLAYING（1），**确认不再卡死**。

### 2) 大屏 + 高清（已优化）
- `fit()` 上限由 760px 提到 **1000px**，并改为**设置画布后备分辨率 = 显示尺寸** + `ctx.setTransform(尺寸/FIELD)` 缩放 + `imageSmoothingEnabled=false`：逻辑坐标仍是 416，但按像素 1:1 渲染，**放大后不再糊**（像素质感更锐利）。
- 机柜外壳 `.frame` 宽度上限由 820px 提到 **1080px**，避免大画布被裁切。
- 帧率：游戏循环本就是 `requestAnimationFrame` 渲染 + 固定步长 60 逻辑更新（与显示器刷新率同步，主流 60Hz、高刷屏可达 120Hz+）。之前"卡顿"完全来自上面的冻结 Bug，修好后即恢复顺滑；本次另把同屏敌人上限 `maxOnField` 4→5、单关敌人数 `stageEnemyCount` 20→24，节奏更紧凑、更"热闹"。

### 3) 地图内容更丰富（已重做 4 关）
- 4 张关卡全部重写为 13x13 字符串模板，密度与变化明显高于旧版：
  - **关卡1 砖阵**：密集砖墙 + 钢柱 + 草丛，四边留可通行走廊。
  - **关卡2 钢铁堡垒**：钢墙内核 + 中央水池 + 砖外墙。
  - **关卡3 钢砖迷宫**：钢/砖棋盘交错，多条通道。
  - **关卡4 十字水域**：中央十字水带 + 砖臂 + 草丛角。
- 基地（鹰巢）仍由 `makeBase` 自动包裹砖墙保护；出生点/玩家位保持空地，确保可玩。
- 验证：重做后加载→START→连跑 + 强制击杀 + 射击 + 触屏 + 暂停 + resize，全程零异常。
