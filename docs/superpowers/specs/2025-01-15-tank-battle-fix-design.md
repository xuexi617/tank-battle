# 坦克大战 Bug 修复设计方案

## 概述

修复经典坦克大战 HTML5 游戏中导致无法正常游玩的 5 个关键 Bug，确保游戏可玩、流程完整。

## 影响范围

- `src/main.js` — 4 处代码修改
- `index.html` — 编码修复

## Bug 1: 坦克炮管方向错乱

**严重程度**: 致命

**根因**: `makeTankSprite(tp, dir)` 根据 `dir` 参数在精灵图上画出炮管朝向（上/右/下/左），然后 `drawTank` 再按 `dir` 旋转画布并绘制，造成双重旋转。

**修复方案**: 
- `makeTankSprite` 始终以 `dir=0`（炮管朝上）绘制，缓存键改为 `tp` 或 `tp_0`
- `drawTank` 保留全部旋转逻辑不变

**涉及函数**: `makeTankSprite`、`drawTank`

## Bug 2: 中间出生点可能被墙堵死

**严重程度**: 严重

**根因**: 地图生成 `mkMp` 清空了 4 个 3×3 区域（左上/右上/左下/右下），但敌人中间出生点 `(192,16)` = 格 `(12,1)` 不在清空范围内。

**修复方案**: 在清空列表中加入 `[0, 12]`（行 0-2，列 12-14），覆盖中间出生点。

**涉及函数**: `mkMp`

## Bug 3: HTML 中文显示乱码

**严重程度**: 中等

**根因**: `index.html` 的中文字符因编码不匹配显示为乱码。

**修复方案**: 将所有中文文本用正确 UTF-8 重新写入，包括标题、按钮文字、按键说明。

**涉及文件**: `index.html`

## Bug 4: 钢墙可被任意子弹摧毁

**严重程度**: 中等

**根因**: 子弹碰撞代码 `if(t.tp<3)` 将砖墙(1)和钢墙(2)等同处理。

**修复方案**: 改为 `if(t.tp===1)` 仅砖墙可摧毁；钢墙(`tp===2`)在简化版中设为不可摧毁。

**涉及函数**: `up` 中的子弹碰撞循环

## Bug 5: 坦克移动可能被卡住

**严重程度**: 中等

**根因**: `mTank` 先检查原始位置→再对齐→再检查对齐位置。对齐位置不合法时直接不移动，即使原始位置合法。

**修复方案**: 改变顺序——先对齐，对齐失败则回退到原始位置再检查。

**涉及函数**: `mTank`

## 实施步骤

1. 修改 `makeTankSprite` 缓存键和绘制逻辑（去掉方向参数影响）
2. 修改 `mkMp` 增加中间出生点清空
3. 修复钢墙碰撞判断
4. 修复 `mTank` 移动逻辑
5. 修复 `index.html` 编码
6. 构建并验证游戏可运行
